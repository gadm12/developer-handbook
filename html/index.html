Highest Priority

Daily HTML Workflows

These cards group the HTML tasks developers usually perform together: start the page, structure it semantically, build a usable form, add media and navigation, and wire up CSS and JavaScript files.

Start Here

Build a Basic Page

Boilerplate 
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

Start every standalone HTML page with the doctype, language, metadata, title, and one visible heading so the document is valid and immediately readable.

Layout

Structure Content Semantically

Semantics 
<header>
  <h1>My Website</h1>
</header>
<nav>...</nav>
<main>
  <section>...</section>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

Use semantic elements when they match the meaning of the content instead of wrapping the whole page in generic <div> elements.

User Input

Create Forms

Forms 
<form action="/submit" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required />

  <label for="message">Message</label>
  <textarea id="message" name="message" rows="5"></textarea>

  <button type="submit">Send</button>
</form>

Match every label to an input, use meaningful name values, and set button types explicitly so the form behaves predictably.

Navigation and Media

Add Images and Links

Media 
<a href="./about.html">About</a>
<a href="#contact">Contact</a>
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  Visit Example
</a>
<img src="./assets/pikachu.png" alt="Pikachu standing and smiling" />

Use relative paths for local pages and assets, secure external links that open new tabs, and always provide useful image alternative text.

Assets

Connect CSS and JavaScript

Files 
<head>
  <link rel="stylesheet" href="./assets/style.css" />
</head>
<body>
  ...
  <script src="./main.js"></script>
</body>

Link CSS in the <head> and load your script near the end of the <body> so the page structure is available before JavaScript runs.

Basic HTML Page

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Page Title</title>
  <link rel="stylesheet" href="./assets/style.css" />

  <h1>Hello World</h1>
  <script src="./main.js"></script>

</html>

<!doctype html>

Tells the browser to use modern HTML.

lang="en"

Identifies the page's main language.

<head>

Contains page metadata, title, and linked files.

<body>

Contains the visible page content.

Meaningful Structure

Semantic Layout

Semantic elements describe the purpose of their content.

Element Purpose
<header>Introductory content
<nav>Navigation links
<main>The page's main content
<section>A related group of content
<article>Independent content
<aside>Supporting or related content
<footer>Footer or closing information
Prefer semantic elements when an element accurately describes the content. Use <div> when no semantic element fits.

Forms

Basic Form

<form>

    name="username"

Element Information

Useful Attributes

class

<div class="card"></div>

Used for reusable CSS styling and groups of elements.

id

<h2 id="projects">Projects</h2>

Used for unique elements, page links, labels, or JavaScript selection.

name

<input type="text" name="pokemonName" />

Determines the form-data key when a form is submitted.

value

<input type="radio" name="difficulty" value="easy" />

Provides the submitted value for a form control.

placeholder

<input type="text" placeholder="Enter Pokemon name" />

Shows temporary guidance inside an empty input.

required

<input type="email" name="email" required />

Prevents submission while the control is empty or invalid.

disabled

<button type="submit" disabled>Submit</button>

Prevents the user from interacting with the element.

data-*

<button data-pokemon-id="25">Remove Pikachu</button>

Stores custom information for JavaScript.

Reading a Data Attribute With JavaScript

const id = button.dataset.pokemonId;
An id should be unique on the page. A class can be reused.

Reusable Template

HTML Starter Template

Copy this structure when beginning a small HTML, CSS, and JavaScript project. 

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My Project</title>
  <link rel="stylesheet" href="./assets/style.css" />
</head>
<body>
  <header>
    <h1>My Project</h1>
    <nav>
        <a href="#home">Home</a>
        <a href="#projects">Projects</a>
        <a href="#contact">Contact</a>
    </nav>
  </header>

  <main>
    <section id="home">
      <h2>Welcome</h2>
      <p>Welcome to my website.</p>
    </section>

    <section id="projects">
      <h2>Projects</h2>
      <article class="card">
        <h3>Project Name</h3>
        <p>Project description.</p>
        <a href="#">View Project</a>
      </article>
    </section>

    <section id="contact">
      <h2>Contact</h2>
      <form>
        <label for="name">Name</label>
        <input id="name" name="name" type="text" required />

        <label for="email">Email</label>
        <input id="email" name="email" type="email" required />

        <label for="message">Message</label>
        <textarea id="message" name="message" rows="5"></textarea>

        <button type="submit">Send</button>
      </form>
    </section>
  </main>

  <footer>
    <p>&copy; 2026 Mohamed Gad</p>
  </footer>

  <script src="./main.js"></script>
</body>
</html>
