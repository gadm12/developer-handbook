A quick reference for building images, running containers, checking logs, opening container shells, and cleaning Docker resources.

Most Used

Quick Commands

These are the Docker commands you are most likely to use during normal development.

Image

Build an Image

Build
docker build -t react-img .

Build an image from the Dockerfile in the current folder.

Container

Run a Vite App

Run
docker run --rm -it -p 5173:5173 react-img

Run the React container and publish Vite on port 5173.

Status

Running Containers

List
docker ps

Display containers that are currently running.

Status

All Containers

List
docker ps -a

Display running and stopped containers.

Output

Follow Logs

Logs
docker logs -f <CONTAINER_ID>

Watch the container output update live.

Terminal

Open Bash

Shell
docker exec -it <CONTAINER_ID> bash

Open an interactive Bash shell inside a running container.

Container

Stop a Container

Stop
docker stop <CONTAINER_ID>

Stop a running container safely.

Cleanup

Remove Unused Resources

Clean
docker system prune

Remove stopped containers, unused networks, and dangling images.

Image Management

Images

Build an image and give it a name.

docker build -t react-img .

The final dot tells Docker to use the current folder as the build context.

List local images.

docker images

Remove an image by name.

docker rmi react-img

Remove an image by ID.

docker rmi <IMAGE_ID>

Force-remove an image.

docker rmi -f <IMAGE_ID>
Force-removing an image may also affect containers that depend on it.

Container Management

Containers

Run a container from an image.

docker run react-img

Run a container and remove it automatically when it stops.

docker run --rm react-img

Run interactively with terminal access.

docker run --rm -it react-img

Run a Vite React application.

docker run --rm -it -p 5173:5173 react-img

Run in the background.

docker run -d react-img

Give the container a name.

docker run --name react-container react-img

List running containers.

docker ps

List running and stopped containers.

docker ps -a

Stop

docker stop <CONTAINER_ID>

Start

docker start <CONTAINER_ID>

Restart

docker restart <CONTAINER_ID>

Remove

docker rm <CONTAINER_ID>

Networking

Ports

Publish a container port to your computer.

docker run -p 5173:5173 react-img
Computer port : Container port

Use a different port on your computer.

docker run -p 5174:5173 react-img

In this example, open:

http://localhost:5174
The app still uses port 5173 inside the container.

Local Files

Volumes and Bind Mounts

Run a container with the current project folder mounted inside /app. 

docker run --rm -it \
```

-v "$(pwd):/app" 
-p 5173:5173 
react-img
```

Keep node_modules inside the container instead of replacing it with the mounted project folder. 

docker run --rm -it \
```

-v "$(pwd):/app" 
-v /app/node_modules 
-p 5173:5173 
react-img
```
A bind mount lets the container see changes you make to files on your computer.

List Docker volumes.

docker volume ls

Remove an unused volume.

docker volume rm <VOLUME_NAME>

Container Output

Logs

Current Output

View Logs

docker logs <CONTAINER_ID>

Display output that the container has already produced.

Live Output

Follow Logs

docker logs -f <CONTAINER_ID>

Keep the terminal open and show new output as it appears.

Show only the most recent lines.

docker logs --tail 50 <CONTAINER_ID>

Container Terminal

Shell

Bash

Open Bash

docker exec -it <CONTAINER_ID> bash

Use this when Bash is installed inside the container.

Shell

Open sh

docker exec -it <CONTAINER_ID> sh

Use this for smaller images that do not include Bash.

Run one command inside a container without opening a full shell.

docker exec <CONTAINER_ID> ls

Remove Resources

Cleanup

Remove One Container

docker rm <CONTAINER_ID>

Remove Stopped Containers

docker container prune

Remove Unused Images

docker image prune

Remove Unused Volumes

docker volume prune

General Cleanup

docker system prune

More Complete Cleanup

docker system prune -a
Review the confirmation message before accepting a prune command. Docker may remove resources you intended to keep.

Multiple Services

Docker Compose

Start Services

docker compose up

Run in Background

docker compose up -d

Stop Services

docker compose down

Rebuild and Start

docker compose up --build

View running Compose services.

docker compose ps

View Compose logs.

docker compose logs

Follow Compose logs live.

docker compose logs -f

Information

Useful Commands

Docker Version

docker --version

Docker Information

docker info

Container Processes

docker top <CONTAINER_ID>

Inspect a Container

docker inspect <CONTAINER_ID>

Inspect an Image

docker inspect <IMAGE_ID>

Disk Usage

docker system df

Development Process

Typical Workflow

Build image ↓ Run container ↓ Test application ↓ Stop container ↓ Clean if needed

Build the Image

docker build -t react-img .

Run the Project

docker run --rm -it -p 5173:5173 react-img

Stop the Project

Ctrl + C

Clean Up if Needed

docker system prune
Because the container uses --rm, Docker automatically removes it after you stop it.
