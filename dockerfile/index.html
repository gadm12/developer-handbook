Dockerfile Examples

These two examples can be used as starting points for React and Python projects.

Node

React and Vite

Dockerfile 
FROM node:24
```

WORKDIR /app

COPY . .

RUN npm install

EXPOSE 5173

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]
```

Starts a Vite development server and makes it accessible outside the container.

Python

Python Application

Dockerfile 
FROM python:3.13
```

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

Installs dependencies from requirements.txt and runs main.py.

What Is a Dockerfile?

A Dockerfile is a text file containing instructions Docker uses to build an image.

The file is normally named exactly:

Dockerfile

No file extension is required.

Dockerfile ↓ docker build ↓ Docker image ↓ docker run ↓ Running container

Basic Structure

A basic React and Vite development Dockerfile may look like this:

FROM node:24
```

WORKDIR /app

COPY . .

RUN npm install

EXPOSE 5173

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]
```

FROM

Base Image

Selects the base image that your application will build on.

Node example:

FROM node:24

Python example:

FROM python:3.13

Smaller Node image:

FROM node:24-alpine
Remember: FROM is usually the first instruction in a Dockerfile.

WORKDIR

Working Folder

Sets the working directory inside the image.

WORKDIR /app

After this instruction, later commands operate from /app. 

WORKDIR /app
```

RUN npm install
```
Docker creates the directory if it does not already exist.

COPY

Project Files

Copies files from your computer into the Docker image.

Copy everything:

COPY . .

First dot

The current project folder on your computer.

Second dot

The current WORKDIR inside the image.

Copy one file:

COPY package.json .

Copy dependency files:

COPY package.json package-lock.json ./

Copy a folder:

COPY src ./src

RUN

Build Time

Executes a command while Docker is building the image.

RUN npm install

Python example:

RUN pip install -r requirements.txt

Run multiple commands:

RUN npm install && npm run build
RUN happens during docker build. It does not run every time the container starts.

EXPOSE

Application Port

Documents which port the application uses inside the container.

EXPOSE 5173

Other examples:

EXPOSE 3000
```

EXPOSE 5000
```
EXPOSE does not publish the port to your computer. You still need the -p option when running the container. 
docker run -p 5173:5173 image-name
Computer port : Container port

CMD

Container Start

Specifies the default command that runs when the container starts.

Node example:

CMD ["npm", "run", "dev"]

Vite example:

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]

Python example:

CMD ["python", "main.py"]

Preferred format:

CMD ["command", "argument1", "argument2"]
A Dockerfile should normally contain only one CMD.

RUN vs CMD

Build Time

RUN

RUN npm install

Installs packages or prepares the image while it is being built.

Start Time

CMD

CMD ["npm", "run", "dev"]

Starts the application when a new container begins running.

ENV

Environment Variables

Creates an environment variable inside the image.

ENV NODE_ENV=development

Multiple variables:

ENV HOST=0.0.0.0
```

ENV PORT=5173
```

Reading an environment variable in Python:

import os
```

port = os.getenv("PORT")
```
Do not place passwords, API tokens, or private keys directly inside a Dockerfile.

.dockerignore

A .dockerignore file prevents unnecessary files from being copied into the image.

Node

.dockerignore 
node_modules
```

```

Python

.dockerignore 
__pycache__
```

*.pyc
.venv
venv

.pytest_cache

```

Benefits

  • Smaller build context
  • Faster builds
  • Avoids copying local dependencies
  • Avoids copying private files accidentally

Build and Run

Build an image

docker build -t my-image .

Build without cache

docker build --no-cache -t my-image .

Run an image

docker run my-image

Remove the container afterward

docker run --rm my-image

Run interactively

docker run --rm -it my-image

Publish a port

docker run --rm -p 5173:5173 my-image

Use a different Dockerfile

docker build -f Dockerfile.dev -t my-image .

Recommended Pattern

Improved Node Dockerfile

Copying the package files before the rest of the project improves Docker layer caching. 

FROM node:24
```

WORKDIR /app

COPY package.json package-lock.json ./

RUN npm install

COPY . .

EXPOSE 5173

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]
```
Package files unchanged ↓ Reuse npm install layer ↓ Faster rebuild
When package.json or package-lock.json changes, Docker runs npm install again.

Common Problems

Vite cannot open in the browser

Make sure Vite listens on every network interface.

CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]

Then publish the port:

docker run -p 5173:5173 react-img

Port is already in use

Use a different port on your computer:

docker run -p 5174:5173 react-img

Open http://localhost:5174. The application still uses port 5173 inside the container.

Changes do not appear

Possible causes:

  • The image was not rebuilt.
  • The project folder was not mounted.
  • The bind mount points to the wrong folder.
  • The container is running an older image.
docker build -t react-img .

Package missing inside the container

After installing a new package, rebuild the image:

npm install react-bootstrap bootstrap
```

docker build -t react-img .
```

Wrong build context

The final dot tells Docker to use the current directory as the build context. 

docker build -t react-img .

If the project is inside a client folder:

docker build -t react-img ./client
