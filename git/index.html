Initialize

Initialize a Git repository inside the current folder.

git init

Check that Git was initialized successfully.

git status

Status

Check the current state of your repository.

git status

The status command shows:

  • Modified files
  • Staged files
  • Untracked files
  • The current branch
  • Whether your branch is ahead or behind

Display a shorter version.

git status --short
Symbol Meaning
M Modified
A Added
D Deleted
?? Untracked

Add

Stage one file.

git add file.py

Stage several files.

git add app.py README.md

Stage everything.

git add .

Stage all changes, including deleted files.

git add --all

Commit

Create a commit with a meaningful message.

git commit -m "Add Pokemon search feature"

Commit tracked files without running git add first.

git commit -am "Fix navigation bug"
Note: git commit -am does not include new, untracked files.

Change the most recent commit message.

git commit --amend -m "New commit message"

Add a forgotten file to the most recent commit.

git add forgotten-file.py
git commit --amend --no-edit

Push

Push committed changes.

git push

Push a branch for the first time.

git push -u origin main

Push a new feature branch.

git push -u origin feature-name

Force push more safely.

git push --force-with-lease
Avoid force pushing unless you understand how it can overwrite remote work.

Pull

Download and merge remote changes.

git pull

Pull from a specific branch.

git pull origin main

Download remote changes without merging.

git fetch

Pull using rebase.

git pull --rebase

Log

View the full commit history.

git log

Press q to exit the log.

View a compact history.

git log --oneline

Display commits as a branch graph.

git log --oneline --graph --all

View the five most recent commits.

git log -5

View commits for one file.

git log -- file.py

Branch

List local branches.

git branch

List local and remote branches.

git branch --all

Create a branch.

git branch feature-name

Create a branch and switch to it.

git switch -c feature-name

Switch branches.

git switch main

Delete a merged branch.

git branch -d feature-name

Force-delete an unmerged branch.

git branch -D feature-name

Merge

Switch to the branch that should receive the changes, then merge the feature branch. 

git switch main
git merge feature-name

A common merge workflow:

git switch main
git pull
git merge feature-name
git push

Merge Conflicts

Git marks conflicting code like this:

<<<<<<< HEAD
Code from the current branch
=======
Code from the incoming branch
>>>>>>> feature-name

Resolve the conflict, remove the markers, and then stage and commit the corrected files. 

git add .
git commit -m "Resolve merge conflict"

Restore / Undo

Unstage one file while keeping its changes.

git restore --staged file.py

Unstage everything.

git restore --staged .

Discard uncommitted changes in one file.

git restore file.py
This permanently removes the file's uncommitted changes.

Undo the most recent commit but keep the changes staged.

git reset --soft HEAD~1

Undo the most recent commit and unstage the changes.

git reset HEAD~1

Safely reverse a commit that was already pushed.

git revert COMMIT_HASH

Preview untracked files that Git can remove.

git clean -n

Remove untracked files.

git clean -f

Stash

Temporarily save unfinished tracked changes.

git stash

Save a stash with a description.

git stash push -m "Work in progress"

Include untracked files.

git stash -u

List stashes.

git stash list

Restore the newest stash and remove it from the stash list.

git stash pop

Restore the newest stash without removing it.

git stash apply

Diff

View unstaged changes.

git diff

View staged changes.

git diff --cached

Compare two branches.

git diff main..feature-name

View changes for one file.

git diff file.py

Show changes from the most recent commit.

git show

Remote

View remote repositories.

git remote -v

Add a remote repository.

git remote add origin REPOSITORY_URL

View information about the origin remote.

git remote show origin

Change the remote URL.

git remote set-url origin NEW_REPOSITORY_URL

Remove a remote.

git remote remove origin

Clone

Copy a GitHub repository to your computer.

git clone REPOSITORY_URL

Clone into a custom folder name.

git clone REPOSITORY_URL custom-folder

Clone one specific branch.

git clone --branch branch-name REPOSITORY_URL

Typical Workflow

Working on Main

git status ↓ git add . ↓ git commit ↓ git pull ↓ git push 
git status
git add .
git commit -m "Describe your changes"
git pull
git push

Feature Branch Workflow

git switch main
git pull
git switch -c feature-name
git add .
git commit -m "Add new feature"
git push -u origin feature-name

After the Pull Request Is Merged

git switch main
git pull
git branch -d feature-name
